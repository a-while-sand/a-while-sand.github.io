下载ElasticSearch

在es官网【https://elasticsearch.cn/download/】找到相关版本，如下载8.5.3

wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.5.3-linux-x86_64.tar.gz

解压并放到/usr/local/目录下

tar -zxvf elasticsearch-8.5.3-linux-x86_64.tar.gz
mv  elasticsearch-8.5.3 /usr/local/

修改配置文件

修改config/elasticsearch.yml
# Cluster
cluster.name: my-es
# Node
node.name: node-1
# Paths
path.data: /usr/local/elasticsearch-8.5.3/data
path.logs: /usr/local/elasticsearch-8.5.3/logs
# Network
network.host: 0.0.0.0
http.port: 9222
# Discovery
cluster.initial_master_nodes: ["node-1"]
# SECURITY
xpack.security.enabled: false
xpack.security.http.ssl:
  enabled: false
修改config/jvm.options
# 若内存小则修改
-Xms128m
-Xmx128m

启动ElasticSearch

/usr/local/elasticsearch-8.5.3/bin/elasticsearch

常见问题

启动需要独立的账号：

org.elasticsearch.bootstrap.StartupException: java.lang.RuntimeException: can not run elasticsearch as root

elasticsearch启动需要独立的账号

useradd esuser
chown -R esuser:esuser /usr/local/elasticsearch-8.5.3

再切换到esuser用户，运行bin/elasticsearch

bootstrap check failure [1] of [3]:

max file descriptors [4096] for elasticsearch process is too low, increase to at least [65535]

bootstrap check failure [2] of [3]: max number of threads [3802] for user [esuser] is too low, increase to at least [4096]

vi /etc/security/limits.conf
* soft nofile 65536
* hard nofile 131072
* soft nproc 4096
* hard nproc 4096
**bootstrap check failure [3] of [3]: **

max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]

vi /etc/sysctl.conf 
vm.max_map_count = 655360
sysctl -p
