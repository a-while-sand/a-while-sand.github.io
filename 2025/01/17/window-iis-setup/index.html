默认IISExpress

只能通过localhost访问程序，若通过IP访问，即使是127.0.01访问也报错

400. The request hostname is invalid.
在操作系统任务栏的状态栏中,选择IISPress站点,右键选择显示所有应用程序

单击站点名称,在下面提示的路径中选择配置路径,打开applicationhost.config文件

找到IISExpress的配置文件并打开applicationhost.config，找到如下代码

找到site的name为【Shou.EDAS.Portal】,在localhost后复制一行，修改成IP,如下
<binding protocol="http" bindingInformation="*:30000:localhost" />
<binding protocol="http" bindingInformation="*:30000:172.16.20.248" />

按上面改为后，当你通过IP地址访问时还是可能会出现400错误，如果出现，就采用下面的方法：

（1）以管理员身份打开CMD命令窗口

（2）输入如下命令：

netsh http add urlacl url=http://172.16.20.248:31173/ user=everyone

上面改的IP和端口号对应，如果报错请确认是否是以管理员身份运行的CMD。

通过命令，可以查看到已经添加进去的URL了

netsh http show urlacl

完成之后重启IISExpress，此时应该就可以使用IP地址访问了。

完成上面两部之后，由于防火墙的原因，其他的电脑可能还无法访问你的服务器，如果出现这种情况，关闭防火墙

（1）控制面板（查看方式选择类别）->系统和安全->Windows防火墙，在左侧选择高级设置
（2）右击入站规则->新建规则->端口->特定本地端口处输入31171->允许连接->3个选项全选->输入新规则名称->完成

用完了也可以将其删除：

netsh http delete urlacl url=http://172.16.20.248:31173/
