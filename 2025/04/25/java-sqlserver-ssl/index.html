Java 驱动程序无法通过SSL加密与SQLServer建立连接

By 不合时宜的人

发表于 2025-04-25

文章目录

JAVA低版本的驱动连接SQLServer时，有可能是英文的报错，

The driver could not establish a secure connection to SQL Server by using Secure Sockets Layer (SSL) encryption. Error: “The server selected protocol version TLS10 is not accepted by client preferences [TLS12]”. ClientConnectionId:d5fd8d69-ae88-4055-9f6d-6e8515224ce2

这个问题其实最主要还得看后面那一句话

The server selected protocol version TLS10 is not accepted by client preferences [TLS12]】，意思是数据库服务端的协议版本是TLS10，而客户端协议版本是TLS12。

只需要在你的设备上找到jdk的安装目录【JAVA_HOME\jre\lib\security】下的java.security改改安全协议的配置即可。删掉这两个：TLSv1、TLSv1.1。特别要注意的是，如果你是选择了jre的运行环境来运行，那就是这个目录【\JAVA_HOME\lib】。

若直接修改java.security，可能会影响其它环境，一般我们复制一份，单独修改，再将项目启动参数配置就可。

"-Djava.security.properties=../my.java.security"
