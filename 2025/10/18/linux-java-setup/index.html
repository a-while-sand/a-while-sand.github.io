Ceonos7.9中设置java应用自启动

使用 systemd 创建自定义服务

假设您的 Java 应用部署目录为 /opt/myapp，其中包含一个名为 myapp.jar 的可执行 JAR 包。

第1步：创建服务单元文件

  1. 使用 root 用户或拥有 sudo 权限的用户登录。
  2. 在 /etc/systemd/system/ 目录下创建一个新的服务文件，例如 myapp.service
vi /etc/systemd/system/myapp.service 

[Unit]
Description=myapp Boot Application
After=network.target

[Service]
Type=forking
ExecStart=/opt/myapp/server/start.sh
ExecStop=/opt/myapp/server/stop.sh
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
start.sh
#!/bin/sh

nohup java ${JAVA_OPT} -jar /opt/myapp/server/myapp.jar --spring.config.location=/opt/myapp/server/config/application.properties >>/opt/myapp/server/logs/${APPLICATION}.log&
stop.sh
#!/bin/sh

APPLICATION=myap
PID=$(cat ${APPLICATION}.pid)
JAVA_OPT="-server"

if [ -z "$PID" ]
then
    echo ${APPLICATION} is already stopped
else
    echo kill  ${PID}
    kill -9 ${PID}
    echo ${APPLICATION} stopped successfully
fi

第2步：设置文件权限并重载 systemd

  1. 给服务文件设置正确的权限。

    chmod 644 /etc/systemd/system/myapp.service

  2. 重新加载 systemd 配置，使其识别新的服务。

    systemctl daemon-reload

第3步：测试和启用服务

  1. 启动服务，进行测试。

    systemctl start myapp

  2. 检查服务状态，查看是否启动成功以及是否有错误日志。

    systemctl status myapp

    如果状态不正常，可以使用 journalctl 查看详细日志：

    journalctl -u myapp -f

  3. 测试停止服务。

    systemctl stop myapp

    再次检查状态，确认已停止。

  4. 设置开机自启。如果以上测试都成功了，就可以启用开机自动启动。

    systemctl enable myapp

    您会看到类似 Created symlink from ... to ... 的提示，表示成功。

第4步：常用管理命令

# 启动服务
 systemctl start myapp

# 停止服务
 systemctl stop myapp

# 重启服务
 systemctl restart myapp

# 查看服务状态
 systemctl status myapp

# 查看服务日志
 journalctl -u myapp

# 实时跟踪日志
 journalctl -u myapp -f

# 禁用开机自启（如果需要）
 systemctl disable myapp
